**Phase:** Discovery
**Difficulty:** 2/10
**Concept:** Two-word constraint intersection

## The Puzzle Mechanism

Players must understand that combining two Limn words NARROWS meaning through intersection.

`sol` alone = vast region (everything solid)
`aqu` alone = vast region (everything water-related)
`sol aqu` = only things that are BOTH solid AND water-related

The intersection includes:
- Ice (frozen water)
- Glacier
- Frost
- Hail
- Frozen tears
- Icicles
- Permafrost
- Snow (debatable - is it solid?)

Players must articulate this intersection, not just name one example.

## What This Teaches

**Primary Concept:** Constraint intersection - combining Limn words narrows meaning to the overlap.

**Secondary Concepts:**
- Two words together are more specific than either alone
- The intersection might still contain multiple valid referents
- "AND" logic, not "OR" logic

**Grammar Foundation:**
This is the fundamental grammar rule of Limn: adjacent words combine by intersection.

## Design Notes

### Why This Works:
1. **Builds on Puzzle 01:** Same conceptual framework (regions)
2. **New mechanic:** Intersection of two regions
3. **Visual support:** Images make the concept tangible
4. **Clear success criteria:** Can articulate the intersection principle

### Connection to Limn:
This puzzle teaches the CORE grammar rule: adjacent words combine by intersection. Everything else in Limn builds on this.

### Potential Issues:
- Players might just say "ice" and think they're done
  - *Mitigation:* Tiered response system pushes toward deeper understanding
- The "AND" logic might be confused with "OR"
  - *Mitigation:* Hint 2 explicitly addresses this
- Discord channel might get cluttered with discussion
  - *Mitigation:* Separate discussion channel from solve channel

### Success Criteria:
- 70% of Puzzle 01 solvers attempt this
- 50% reach Tier 3 understanding
- Average solve time: 30-60 minutes
- Zero frustration quit (always possible to progress with hints)

## Technical Implementation

### Bot Logic (Pseudocode):

```python
TIER_1_ANSWERS = ["ice", "glacier", "frost", "frozen", "hail", "snow", "icicle"]
TIER_2_INDICATORS = ["multiple", "various", "forms", "different", "all", "both"]
TIER_3_PHRASES = ["intersection", "and", "both constraints", "overlap",
                   "narrow", "combines", "where.*meet"]

def evaluate_puzzle_02(response):
    response_lower = response.lower()

    # Check Tier 3 first
    tier_3_match = any(re.search(phrase, response_lower) for phrase in TIER_3_PHRASES)
    has_examples = any(ans in response_lower for ans in TIER_1_ANSWERS)

    if tier_3_match and has_examples:
        return "TIER_3"

    # Check Tier 2
    tier_2_match = any(ind in response_lower for ind in TIER_2_INDICATORS)
    example_count = sum(1 for ans in TIER_1_ANSWERS if ans in response_lower)

    if tier_2_match or example_count >= 2:
        return "TIER_2"

    # Check Tier 1
    if example_count >= 1:
        return "TIER_1"

    return "INCORRECT"
```
