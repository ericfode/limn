# Alternative Symbols Test: Methodology

**Purpose:** Systematic evaluation of alternative symbols for semantic operations

**Date:** 2026-02-03
**Evaluator:** Claude Code, Linguist Agent

---

## Test Framework

Three semantic operations need symbols because existing operators conflict:

| Operation | Conflict | Need | Goal |
|-----------|----------|------|------|
| Semantic Subtraction | `-` = intensity mild | New symbol for component removal | Refinement/essence |
| Superposition | `~` = delegation + approaching | New symbol for quantum both/and | Uncertainty representation |
| Conditional | `\|` = scope | New symbol for given-context | Contextual dependency |

---

## Evaluation Criteria

Each candidate evaluated on:

1. **Semantic Clarity** (40%)
   - Does the symbol naturally suggest its meaning?
   - Is interpretation intuitive?
   - Are alternative interpretations unlikely?

2. **Technical Fitness** (25%)
   - No conflicts with existing operators
   - Compatible with Limn grammar
   - Composable with other operators
   - Parser complexity reasonable

3. **User Accessibility** (20%)
   - Typability (ASCII vs Unicode)
   - Learnability for non-technical users
   - Consistency with existing Limn patterns
   - Cultural universality

4. **Architectural Alignment** (15%)
   - Fits Limn's hybrid philosophy (symbols + words)
   - Matches operator design principles
   - Consistent with consciousness architecture
   - Logical precedence with other operators

---

## Test Process

### Step 1: Generate Alternatives

For each operation, generated 3 candidates:

**Semantic Subtraction candidates:**
- `÷` - mathematical division (semantic parallel)
- `\` - backslash (exclusion metaphor)
- `without` - natural language (word operator)

**Superposition candidates:**
- `⊗` - tensor product (mathematical grounding)
- `≈` - approximately equal (uncertainty)
- `±` - plus-minus (both values simultaneously)

**Conditional candidates:**
- `/` - forward slash (set theory "such that")
- `given` - natural language (conditional)
- `:` - colon (type theory "has type")

### Step 2: Analyze Each Candidate

For each candidate:
1. **Semantic Assessment** - Does it clearly express the concept?
2. **Rationale** - Why this symbol?
3. **Strengths** - Advantages specific to Limn
4. **Weaknesses** - Disadvantages or conflicts
5. **Examples** - 3 practical examples showing usage

### Step 3: Comparative Evaluation

Candidates evaluated against criteria in context of:
- Limn's existing operator set
- Consciousness architecture requirements
- Natural language integration
- Technical parsing constraints

### Step 4: Recommendation

Each operation: one primary choice + one acceptable alternative

## Precedence Determination

Semantic operators placed in precedence hierarchy:

```
Precedence (highest to lowest):
1. Context boundaries: ⟨⟩
2. Ground truth: ∎
3. Temporal: ∿
4. Semantic operations: without, ±, given
5. Delegation: ~
6. Focus: @
7. Context ops: ⊕ ⊗ ⊂
8. Sequence: →
9. Scope: |
```

Rationale: Semantic operations work on core terms, composition operators work on expressions.
